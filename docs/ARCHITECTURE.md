## 第一章 项目概述与核心目标

### 1.1 项目背景

家装行业长期存在「信息不对称、预算超支、施工不透明、设计与落地脱节」四大核心痛点。「集装修」平台聚焦C端装修用户，以AI技术为核心驱动力，打造「验房-设计-预算-施工-监工-验收-维保」全流程一站式装修服务平台。

### 1.2 核心量化目标

| 指标 | 目标值 |
|------|---------|
| 核心服务可用性 | ≥99.95% |
| 注册用户数 | ≥10,000（3个月） |
| 用户满意度 | ≥85% |
| 付费转化率 | ≥5% |
| 预算误差率 | ≤5% |
| AI识别准确率 | ≥90% |

## 第二章 六层核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                        接入层                               │

                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      API网关层 (Nginx + APISIX)            │
│          统一鉴权 │ 流量管控 │ 请求路由 │ 限流熔断        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      应用服务层 (微服务集群)                │
│  用户服务  │  验房服务  │  设计服务  │  预算引擎  │  监工服务  │
│           (Go)          │       (Python FastAPI)          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      AI能力层                               │
│    通用AI适配层 │ 家装垂直模型 │ 自研核心引擎 │ 模型管理  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      数据存储层                             │

                              ↓

```

## 第三章 核心业务模块（8大模块）

### 3.1 用户全域管理模块

**定位**：全平台基础底座

| 子模块 | 功能 |
|--------|------|
| 全角色账号管理 | 业主、工长、供应商、运营 |
| 统一身份认证 | JWT+OAuth2.0，单点登录 |
| 精细化权限管控 | RBAC+ABAC混合模型 |
| 用户标签体系 | 精准推荐、个性化服务 |
| 房屋档案管理 | 业主多套房档案管理 |
| 操作审计 | 全链路日志，6个月留存 |

### 3.2 AI智能验房模块

**定位**：装修全流程起点

| 子模块 | 功能 |
|--------|------|
| 图片采集预处理 | 畸变矫正、去噪、增强 |
| AI问题识别 | 8类问题识别，准确率≥90% |
| 报告生成导出 | PDF下载、微信分享 |
| 人工复核 | 高风险问题100%复核 |

**8类问题识别**：
1. 墙面开裂
2. 地面不平
3. 防水渗漏
4. 门窗密封
5. 水电隐患
6. 墙面空鼓
7. 阴阳角不垂直
8. 下水不畅

**验收标准**：
- 识别准确率≥90%
- 单张响应≤2秒
- 报告生成≤10秒
- 可用性≥99.95%

### 3.3 AI设计与预算联动模块（核心壁垒）

**定位**：差异化核心壁垒

| 子模块 | 功能 |
|--------|------|
| 户型识别建模 | OCR+视觉，≥95%准确率 |
| 设计方案生成 | 3套差异化方案，≤3秒 |
| 效果图渲染 | 10秒高清渲染 |
| 预算联动引擎 | DAG工序链，≤300ms |
| 施工图生成 | 6大核心图纸 |

**DAG工序影响链**：
```
设计元素 → 工序变更 → 材料清单 → 人工成本 → 地区调价 → 实时预算
```

**四大基础库**：
- 材料库（含损耗、市场价、采购价）
- 工序库（工时定额、前后置工序）
- 人工成本库（31省市分地区指导价）
- 地区调价系数库

**验收标准**：
- 户型识别≥95%，≤5秒
- 方案生成≤10秒
- 预算联动≤300ms
- 预算误差≤5%

### 3.4 施工全流程管理模块

**定位**：业务落地核心载体

| 子模块 | 功能 |
|--------|------|
| 项目合同管理 | 电子合同、自动建档 |
| 施工计划生成 | 自动生成节点计划 |
| 进度节点管控 | 进度上报、节点验收 |
| 整改工单 | 问题闭环管理 |
| 材料台账 | 进场验收、记录 |
| 多角色协同 | 业主、工长、监理 |

**8大施工节点**：
1. 开工准备
2. 水电改造
3. 防水工程
4. 泥瓦工程
5. 木工工程
6. 油漆工程
7. 安装工程
8. 竣工验收

### 3.5 AI云监工模块（核心壁垒）

**定位**：无人化云监工

| 子模块 | 功能 |
|--------|------|
| 影像采集 | 照片/视频抽帧、预处理 |
| AI进度识别 | ≥95%准确率 |
| 合规检测 | ≥90%准确率 |
| 预警引擎 | 高/中/低风险分级 |

**双模型冗余**：
- 主模型：百度飞桨

**验收标准**：
- 进度识别≥95%
- 合规识别≥90%
- 安全隐患≥92%
- 单张响应≤3秒

### 3.6 标准化验收模块

**定位**：全流程闭环收尾

## 第四章 核心技术规范

### 4.1 AI能力层架构

```
用户请求 → AI中台适配层 → 模型路由 → 垂直引擎 → 人工兜底 → 返回
```

**模型选型**：
| 能力 | 主模型 | 备用模型 |
|------|--------|----------|
| 对话生成 | 智谱GLM-4.5 | 讯飞星火 |
| 图像识别 | 阿里云视觉 | 腾讯云影像 |
| 预算引擎 | 自研Go引擎 | - |

### 4.2 数据存储选型

| 数据类型 | 存储方案 |
|----------|----------|
| 业务数据 | PostgreSQL |
| 热点缓存 | Redis |
| 大文件 | 腾讯云COS |
| 日志分析 | ELK Stack |
| 向量数据 | Milvus |

### 4.3 接口响应时间标准

| 接口类型 | 目标时间 |
|----------|----------|
| 普通接口 | ≤500ms |
| 预算联动 | ≤300ms |
| AI识别(验房) | ≤2秒 |
| AI生成(效果图) | ≤10秒 |
| 身份认证 | ≤100ms |
