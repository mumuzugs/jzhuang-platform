## 第一章 架构设计总纲

### 1.1 方案核心定位

本架构专为家装垂直场景打造，以AI全链路赋能为核心壁垒，完全匹配「装修新手零门槛一站式服务」的产品定位，彻底解决：
- 传统装修平台技术与业务脱节
- AI功能无行业化落地路径
- 架构无法适配家装低频长周期场景
- 合规与数据安全缺失

实现「业务可闭环、技术可落地，成本可管控、能力可迭代」的商用级目标。

### 1.2 架构设计核心原则

| 原则 | 说明 |
|------|------|
| 业务优先，壁垒先行 | 重点保障「AI设计-预算实时联动引擎、AI验房、AI云监工」三大核心功能 |
| 高可用与弹性扩展 | 支持用户量从0到千万级的平滑扩展，核心服务可用性≥99.95% |
| 合规安全红线前置 | 符合《网络安全法》《个人信息保护法》，满足三级等保要求 |
| AI能力可迭代、可兜底 | 双模型冗余备份，核心AI能力具备人工兜底机制 |
| 低成本快速落地 | 微服务模块化设计，3.5个月完成MVP上线验证 |

## 第二章 整体技术架构总图

### 2.1 六层分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                        接入层                               │
│     微信小程序(Taro)    │    PC管理后台(React)    │   APP(Flutter)   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      API网关层 (Nginx + APISIX)            │
│          统一鉴权 │ 流量管控 │ 请求路由 │ 限流熔断        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      应用服务层 (微服务集群)                │
│  用户服务  │  验房服务  │  设计服务  │  预算引擎  │  监工服务  │
│           (Go)          │       (Python FastAPI)          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      AI能力层                               │
│    通用AI适配层 │ 家装垂直模型层 │ 自研核心引擎 │ 模型管理  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      数据存储层                             │
│  PostgreSQL │ Redis │ 对象存储COS │ 数据仓库 │ 向量数据库 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      基础设施层                             │
│         Docker │ Kubernetes │ 腾讯云 │ 监控告警          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Taro + React + Flutter | 微信小程序 + PC管理后台 + APP |
| API网关 | Nginx + APISIX | 高性能流量网关 |
| 高并发服务 | Go (Gin框架) | 预算引擎、支付服务、网关 |
| AI服务 | Python FastAPI | AI对接、业务快速迭代 |
| 数据库 | PostgreSQL + Redis | 主库 + 缓存 |
| 文件存储 | 腾讯云COS | 效果图、施工图、照片 |
| 容器化 | Docker + Kubernetes | 服务编排 |
| 云厂商 | 腾讯云 | COS、CLB、WAF、云监控 |

### 2.3 核心业务数据流

```
用户请求链路：
用户发起请求 → 接入层 → API网关(鉴权+路由) → 应用服务 → AI能力层 → 返回结果

数据流转链路：
业务数据 → 主库写入 → 缓存同步 → 对象存储(大文件) → 数据仓库(分析) → AI模型迭代

AI能力调用链路：
业务请求 → AI中台适配层 → 模型路由 → 垂直引擎 → 人工兜底校验 → 返回结果
```

## 第三章 各层级详细设计

### 3.1 接入层设计

#### 3.1.1 多端技术选型

| 端 | 技术栈 | 说明 |
|----|--------|------|
| 微信小程序 | Taro + React | 主包2M以内，分包加载 |
| PC管理后台 | React + Ant Design Pro | PC端管理界面 |
| 移动APP | Flutter | iOS + Android 双端 |
| H5 | React | 移动端浏览器适配 |

#### 3.1.2 性能规范

- 小程序主包体积 ≤ 2M
- APP首屏加载 ≤ 2秒
- H5页面加载 ≤ 1.5秒
- 页面切换无卡顿
- 核心功能操作步骤 ≤ 3步

### 3.2 API网关层设计

#### 核心技术选型
- **Nginx**：七层负载均衡、SSL终端
- **APISIX**：API网关、流量管控、限流熔断

#### 核心功能
| 功能 | 说明 |
|------|------|
| 统一鉴权 | JWT Token验证、OAuth2.0 |
| 流量管控 | 限流、熔断、降级 |
| 请求路由 | 动态路由、服务发现 |
| 日志审计 | 全链路日志、可追溯 |
| 安全防护 | WAF、防刷、防爬 |

### 3.3 应用服务层设计

#### 微服务拆分（DDD领域驱动）

| 微服务 | 语言 | 职责 |
|--------|------|------|
| 用户服务 | Go | 用户体系、VIP管理 |
| 验房服务 | Python | AI验房、报告生成 |
| 设计服务 | Python | AI设计、效果图 |
| 预算引擎 | Go | 预算计算、联动引擎 |
| 施工服务 | Python | 施工计划、节点管理 |
| 监工服务 | Python | 云监工、进度识别 |
| 支付服务 | Go | 订单、微信支付 |

#### 服务间通信
- **gRPC**：高性能跨语言调用
- **HTTP**：对外REST API

### 3.4 AI能力层设计（核心壁垒）

#### 架构设计
```
┌─────────────────────────────────────────┐
│          AI能力中台统一适配层            │
│    (统一接口、路由、监控、故障切换)       │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           家装垂直AI模型层               │
│  智谱GLM-4.5 │ 阿里云视觉 │ 讯飞星火   │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           自研核心引擎层                  │
│  预算联动引擎 │ 户型识别 │ 进度识别      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│            模型管理平台                   │
│  效果评估 │ 版本管理 │ 灰度发布         │
└─────────────────────────────────────────┘
```

#### 三大核心壁垒

| 功能 | 技术方案 | 响应时间 |
|------|----------|----------|
| AI验房 | 阿里云视觉 + 家装垂直微调 | ≤10秒 |
| AI设计+预算联动 | 智谱GLM + 自研联动引擎 | ≤300ms |
| AI云监工 | 阿里云视觉 + 进度识别引擎 | ≤15秒 |

### 3.5 数据存储层设计

#### 多模数据存储选型

| 数据类型 | 存储方案 | 说明 |
|----------|----------|------|
| 业务数据 | PostgreSQL | 用户、订单、设计方案 |
| 热点缓存 | Redis | Token、会话、热点数据 |
| 大文件 | 腾讯云COS | 效果图、施工图、照片 |
| 日志分析 | ELK Stack | 全链路日志 |
| 向量数据 | Milvus | AI特征向量存储 |

#### 数据安全策略

| 级别 | 数据类型 | 策略 |
|------|----------|------|
| 核心敏感 | 密码、支付信息 | AES-256加密、单独存储 |
| 一般敏感 | 手机号、地址 | 脱敏存储、权限管控 |
| 普通数据 | 公开内容 | 标准存储 |

## 第四章 核心业务模块技术实现

### 4.1 AI验房模块

#### 技术架构
```
用户上传照片 → API接收 → 图片预处理 → 阿里云视觉识别 → 垂直引擎处理 → 报告生成 → 返回
```

#### 8类问题识别

| 问题类型 | 识别方案 | 准确率目标 |
|----------|----------|-------------|
| 墙面开裂 | 图像分割 + 裂缝检测 | ≥90% |
| 地面不平 | 深度学习 + 3D重建 | ≥90% |
| 防水渗漏 | 图像分类 + 时序分析 | ≥90% |
| 门窗密封 | 边缘检测 + 红外 | ≥85% |
| 水电隐患 | 规则引擎 + 图像识别 | ≥90% |
| 墙面空鼓 | 声波检测 + 图像 | ≥85% |
| 阴阳角不垂直 | 几何检测 | ≥90% |
| 下水不畅 | 视频分析 | ≥85% |

### 4.2 AI设计+预算联动模块（核心壁垒）

#### 预算联动引擎架构
```
设计参数变更 → 联动引擎 → 工序映射 → 材料计算 → 人工核算 → 地区调价 → 实时预算
```

#### 全链路映射模型
```
设计元素 → 施工工序 → 材料清单 → 人工定额 → 地区系数 → 实时报价
```

### 4.3 AI云监工模块

#### 技术架构
```
工地照片上传 → AI进度识别 → 施工节点匹配 → 异常检测 → 档案归档 → 通知推送
```

## 第五章 安全与合规设计

### 5.1 合规体系

| 法规 | 要求 | 落地措施 |
|------|------|----------|
| 《网络安全法》 | 网络安全保护 | 三级等保认证 |
| 《个人信息保护法》 | 数据最小化 | 最小必要原则 |
| 《数据安全法》 | 数据分类分级 | 分级存储策略 |
| 《生成式AI服务管理》 | AI内容安全 | 人工兜底审核 |

### 5.2 安全防护

- 全站HTTPS加密传输
- SQL注入防护（参数化查询）
- XSS防护（输入过滤）
- CSRF Token验证
- 接口防刷（限流+验证码）
- WAF防火墙
